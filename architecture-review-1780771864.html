One deep module, helpers absorbed, schema moved to LLM seam

Problem. The dispatcher in engine.sh is a bare case statement — removing it just moves the case into the run loop.

Solution. Merge result helpers into tools.sh, move schema generation to chat-parser.sh, inline the dispatcher into run-loop.sh.

Wins

  • Leverage: one interface (baish_tool_execute) instead of bouncing between two files
  • Locality: schemas live at the LLM interface seam, not in the execution path
  • Deletion test: the dispatcher fails — its removal moves complexity, doesn't eliminate it
  • Delete 1 shallow file (109 lines), spread helpers where they're consumed

Routing seam isolates format; each adapter is independently testable

Problem. _copilot_chat_single branches on model family and inlines two completely different API protocols (Responses vs Chat Completions) in the same function, making it 150+ lines with hard-to-test conditional paths.

Solution. Extract _copilot_chat_responses and _copilot_chat_completions as separate functions; route at the entry point.

Wins

  • Locality: each API format's payload building + response parsing is in one function
  • Leverage: adding a third route (e.g. Anthropic) is a new adapter, not a nested branch
  • Tests: each adapter tested in isolation with mock HTTP, no branching needed
  • Interface shrinks: the routing seam is just a model-prefix regex

Noteworthy. 7 of 17 source modules have no dedicated test file. The run-loop — the central orchestration module — has no tests at all. Session management, skill loading, state persistence, TAB completion, and the tool engine all go untested.

Pattern. The modules with tests are either pure data transformations (chat-parser.sh, errors.sh) or provider adapters with mock HTTP. Modules whose interface is "mutate global arrays and return nothing" (session.sh) or "drive a loop" (run-loop.sh) are harder to test — their current interface offers no seam for substitution.

ADR conflict. Not a refactoring candidate — a test coverage gap. Worth exploring after the candidates above are addressed, since each deepening candidate narrows the untested surface.

Top recommendation

Collapse the tool engine dispatcher

It's the shallowest module in the codebase — a bare case statement with no interface of its own. The helpers it carries belong in tools.sh; the schema generation belongs in chat-parser.sh. Tackle this first because it's the smallest change with the clearest before/after: delete one file, absorb its logic into natural homes, and every future call site crosses one fewer module boundary.
