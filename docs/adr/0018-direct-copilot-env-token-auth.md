# Copilot env auth uses the normal GitHub-to-Copilot token exchange

BAISH V1 will prefer `COPILOT_GITHUB_TOKEN`, then `GH_TOKEN`, then `GITHUB_TOKEN` as GitHub-token inputs for Copilot auth, but it will still follow the normal Copilot flow by exchanging that GitHub token at `/copilot_internal/v2/token` before calling inference endpoints. In this env-token mode BAISH persists metadata-only Copilot auth state under `~/.baish/auth/copilot.json` without storing either the env secret or the exchanged Copilot bearer token, while preserving the existing device-flow path for sessions where no env token is available.
